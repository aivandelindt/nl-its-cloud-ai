I

The evolution of intelligence

An LLM is not a database and not a reasoner with a will. It is a very large neural network trained to predict the next token — a brilliant pattern engine whose knowledge is frozen at a cutoff, with no memory between calls and no ability to touch your systems.

A raw model is a powerful brain in a jar. Tools, context, and a loop turn it into an agent.

LLMPredicts text from patterns in weights
+ RAGRetrieves your docs at query time for grounded answers
+ ToolsCalls functions — read files, run queries, hit APIs
+ MCPStandardizes how tools plug in — build once, use everywhere
AgentCloses the loop: plan, act, observe, adapt until done

What makes a system agentic?

Autonomy across many steps. Tool use with real results. Adaptation from errors and test failures. In Copilot, this manifests as Agent Mode — give a goal, it plans, edits files, runs terminal commands, calls MCP tools, and iterates.

Agentic traits

  • Autonomy
  • Tool use
  • Adaptation
  • Multi-step planning

Copilot modes

  • Completions
  • Chat / Edit
  • Agent Mode ← target
II

Model Context Protocol

MCP is the USB-C for AI — an open standard under the Linux Foundation that replaces N×M bespoke integrations with one protocol. Any MCP-aware host connects to any server via JSON-RPC 2.0 over stdio or HTTP.

3
Roles: Host · Client · Server
3
Primitives: Tools · Resources · Prompts
1
Protocol for all connectors

Before MCP

N clients × M tools = N×M bespoke integrations. Brittle, unscalable, unmaintainable.

With MCP

One protocol. Plug-and-play.

III

The cloud engineer workbench

VSCode with Copilot Agent Mode, OpenTofu CLI, Azure CLI, and Dev Containers — wired together through MCP so the agent reads state, plans changes, and applies them safely in version control.

main.tfHCL
terraform {
  required_version = ">= 1.7.0"
  backend "azurerm" {
    resource_group_name  = "rg-tfstate"
    storage_account_name = "sttfstateteam"
    container_name       = "tfstate"
    key                  = "platform.tfstate"
  }
}
.vscode/mcp.jsonJSON
{
  "servers": {
    "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp/" },
    "azure": { "type": "stdio", "command": "npx", "args": ["-y", "@azure/mcp@latest", "server", "start"] },

Workflow: tofu init → tofu plan → tofu apply. Remote state means agent and team share one source of truth.

V

Building the plugin

Six customization layers bundle into one shippable unit: Instructions, Prompt files, Skills, Custom agents, MCP servers, and Hooks.

Instructions

Always-on rules via applyTo globs

Skills

Auto-discovered when task matches

Agents

Planner → Implementer handoffs

Hooks

Deterministic scripts on lifecycle events — block apply without approved plan

MCP

Live tool connections

Security

Trust sources, Key Vault secrets, human in the loop
