Evolution

What an LLM actually is

An LLM is a very large neural network trained to predict the next token. Everything it "knows" is compressed into weights. It is a brilliant pattern engine — not a database, not a reasoner with a will.

Knowledge cutoff
Frozen at training time
No memory
Between calls
No system access
Cannot touch your infra
Hallucination
Confident but wrong
Takeaway: A raw model is a powerful brain in a jar. Tools, context, and a loop turn it into an agent.

Model Context Protocol

MCP is an open standard (Linux Foundation) defining one common way for AI clients to connect to tools and data. Build a connector once — every MCP-aware host can use it.

Host
VSCode + Copilot — where the user works
Client
One per server connection in the host
Server
Exposes Tools, Resources, Prompts

Transport: JSON-RPC 2.0 over stdio (local) or HTTP/SSE (remote)

OpenTofu + Azure baseline

main.tfHCL
terraform {
  required_version = ">= 1.7.0"
  required_providers {
    azurerm = { source = "hashicorp/azurerm", version = "~> 4.0" }
  }
  backend "azurerm" {
    resource_group_name  = "rg-tfstate"
    storage_account_name = "sttfstateteam"
    container_name       = "tfstate"
    key                  = "platform.tfstate"
  }
}

MCP server wiring

.vscode/mcp.jsonJSON
{
  "servers": {
    "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp/" },
    "azure": { "type": "stdio", "command": "npx",
      "args": ["-y", "@azure/mcp@latest", "server", "start"] },
    "terraform": { "type": "stdio", "command": "docker",
      "args": ["run", "-i", "--rm", "hashicorp/terraform-mcp-server"] }
  }
}

Agent Mode walkthrough

Prompt: "Stand up a dev AKS landing zone and open a PR."

  1. Instructions inject your standards (naming, tagging, region policy)
  2. Planner queries Azure MCP + Terraform MCP for resources and docs
  3. Implementer scaffolds .tf files via the OpenTofu skill
  4. PreToolUse hook runs tofu validate + policy check before apply
  5. GitHub MCP opens a pull request with plan output attached

Custom Copilot plugin

Six layers: Instructions · Prompt files · Skills · Custom agents · MCP servers · Hooks

plugin.jsonJSON
{
  "name": "azure-iac-plugin",
  "displayName": "Azure IaC Copilot",
  "version": "1.0.0",
  "description": "Plan & ship Azure infra with OpenTofu",
  "publisher": "cloud-team",
  "contributes": {
    "agents": ["./agents"], "skills": ["./skills"],
    "instructions": ["./instructions"], "prompts": ["./prompts"],
    "mcpServers": "./.mcp.json", "hooks": "./hooks/hooks.json"
  }
}
Security: Trust MCP sources. Secrets via Key Vault. Human in the loop for destructive ops. Hooks enforce deterministically.
