Section 01

Evolution

An LLM is a pattern engine — not a database, not a reasoner with a will. Everything it knows is compressed into weights. The rest of this site is about giving it eyes, hands, and judgment.

The core idea

A very large neural network trained to predict the next token. A brilliant pattern engine.

Built-in limits

  • Frozen at a knowledge cutoff
  • No memory between calls
  • Cannot touch your systems
  • Can hallucinate confidently

Section 02

MCP — the USB-C for AI

An open standard (Linux Foundation) defining one common way for AI clients to connect to tools and data. Build a connector once — every MCP-aware host can use it.

Before MCP

N clients × M tools = N×M bespoke integrations. Brittle and unscalable.

With MCP

One protocol. N + M connectors. Any client, any server, plug-and-play.

Section 04

A request, end to end

"Stand up a dev AKS landing zone and open a PR."

1Instructions inject your standards (naming, tagging, region policy).
2Planner queries Azure MCP + Terraform MCP for resources and module docs.
3Implementer scaffolds .tf files via the OpenTofu skill.
4PreToolUse hook runs tofu validate + policy check before apply.
5GitHub MCP opens a pull request with plan output attached.

Section 05

Build the plugin

Instructions

Always-on rules via applyTo globs.

Skills

Auto-discovered procedures when relevant.

Custom agents

Planner (read-only) → Implementer (writes + apply).

Hooks

Deterministic guardrails — block apply without approved plan.

plugin.jsonjson
{
  "name": "azure-iac-plugin",
  "displayName": "Azure IaC Copilot",
  "contributes": {
    "agents": ["./agents"], "skills": ["./skills"],
    "instructions": ["./instructions"], "mcpServers": "./.mcp.json",
    "hooks": "./hooks/hooks.json"
  }
}
